# English Grammar Rules Reference

Key grammar rules organized for quick reference during English coaching sessions. Focus on rules most relevant to intermediate (B1-B2) learners in technical/professional contexts.

---

## Verb Tenses Quick Reference

### Present Simple
**Form**: Subject + base verb (+ s/es for 3rd person singular)
**Use**: Habits, facts, schedules, general truths
**Signals**: always, usually, every day, often, sometimes, never
**Example**: "The server restarts every night at midnight."

### Present Continuous
**Form**: am/is/are + verb-ing
**Use**: Actions happening now, temporary situations, future arrangements
**Signals**: now, right now, at the moment, currently, today
**Example**: "I am debugging the authentication module."

### Present Perfect
**Form**: have/has + past participle
**Use**: Past action with present relevance, experience, unfinished time periods
**Signals**: already, just, yet, ever, never, since, for, recently, so far
**Example**: "I have fixed three bugs today." (today isn't over)

### Present Perfect Continuous
**Form**: have/has + been + verb-ing
**Use**: Action started in past, continuing now, emphasis on duration
**Signals**: since, for, all day, how long
**Example**: "I have been working on this feature since Monday."

### Past Simple
**Form**: Subject + past form (regular: -ed; irregular: learned individually)
**Use**: Completed actions at a specific past time
**Signals**: yesterday, last week, in 2020, ago, when I was...
**Example**: "I deployed the update yesterday."

### Past Continuous
**Form**: was/were + verb-ing
**Use**: Action in progress at a specific past time, background actions
**Signals**: while, when, at that time
**Example**: "I was reviewing the code when the server crashed."

### Past Perfect
**Form**: had + past participle
**Use**: Action completed before another past action
**Signals**: before, after, by the time, already, just
**Example**: "I had already committed the fix before you reported the bug."

### Future Simple
**Form**: will + base verb
**Use**: Predictions, spontaneous decisions, promises
**Example**: "I will send you the pull request tonight."

### Future with "going to"
**Form**: am/is/are + going to + base verb
**Use**: Plans, intentions, evidence-based predictions
**Example**: "I am going to refactor the database layer next sprint."

## Articles

### When to Use "a/an"
- First mention of a countable singular noun: "I found **a** bug."
- Non-specific instance: "We need **a** developer."
- With job titles: "She is **an** engineer."
- After "there is": "There is **a** problem."

### When to Use "the"
- Already mentioned: "I found a bug. **The** bug is in the parser."
- Specific/unique: "**The** main branch is protected."
- Shared knowledge: "Can you check **the** logs?" (we both know which logs)
- Superlatives: "This is **the** best solution."
- Ordinals: "**The** first commit."

### When to Use NO Article
- Plural/uncountable nouns used generally: "Bugs are inevitable." "Software requires testing."
- Abstract concepts: "Quality matters."
- Languages: "I speak Russian."
- Most proper nouns: "JavaScript", "Docker", "GitHub"

## Prepositions

### Prepositions of Time
- **at**: precise time → "at 3 PM", "at noon", "at night"
- **on**: days/dates → "on Monday", "on March 5th", "on my birthday"
- **in**: longer periods → "in January", "in 2024", "in the morning", "in 5 minutes"
- **for**: duration → "for 3 hours", "for 2 years"
- **since**: starting point → "since Monday", "since 2020"
- **during**: within a period → "during the meeting"

### Prepositions of Place
- **at**: specific point → "at the office", "at the top"
- **on**: surface → "on the screen", "on the table", "on the server"
- **in**: enclosed space → "in the file", "in the database", "in the list"

### Verb + Preposition (Common in Tech)
- depend **on**, rely **on**, work **on**, focus **on**
- look **at**, look **for**, look **into**
- consist **of**, think **of/about**
- result **in**, result **from**
- refer **to**, relate **to**, respond **to**
- deal **with**, cope **with**, work **with**

## Conditionals

### Zero Conditional (Facts)
**Form**: If + present simple, present simple
**Use**: General truths, always-true conditions
**Example**: "If you push to main, the CI pipeline runs."

### First Conditional (Real/Likely)
**Form**: If + present simple, will + base verb
**Use**: Likely future situations
**Example**: "If the tests pass, I will merge the PR."

### Second Conditional (Unreal/Hypothetical)
**Form**: If + past simple, would + base verb
**Use**: Hypothetical or unlikely situations
**Example**: "If I had more time, I would rewrite the whole module."

### Third Conditional (Past Unreal)
**Form**: If + past perfect, would have + past participle
**Use**: Imagining a different past outcome
**Example**: "If we had written tests, we would have caught the bug earlier."

## Modal Verbs

### Ability
- **can**: present ability → "I can write Python."
- **could**: past ability, polite requests → "I could do it yesterday." / "Could you review this?"

### Obligation
- **must**: strong obligation (speaker's authority) → "You must use HTTPS."
- **have to**: external obligation → "We have to follow the style guide."
- **should**: advice/recommendation → "You should add error handling."

### Possibility
- **might/may**: uncertain possibility → "This might cause a memory leak."
- **could**: theoretical possibility → "This could be the issue."
- **must**: logical deduction → "The server must be down." (I'm almost certain)

## Sentence Structure

### Basic Patterns
- **SVO**: "The function **returns** a value."
- **SVO + adverb**: "The function **always returns** a value."
- **There is/are**: "There **are** three bugs in the module."

### Complex Sentences
- **Because/Since/As** (reason): "I refactored it because it was too complex."
- **Although/Even though** (contrast): "Although it works, the code needs cleanup."
- **So that/In order to** (purpose): "I added logging so that we can debug issues."
- **While/Whereas** (comparison): "Python is dynamic, while Java is static."

### Relative Clauses
- **who/that** (people): "The developer who wrote this left the company."
- **which/that** (things): "The function that handles auth is broken."
- **where** (place): "The file where the config is stored."
- **when** (time): "The day when we deployed the update."

## Common Technical English Patterns

### Passive Voice (Common in Documentation)
**Form**: be + past participle
- "The data **is stored** in PostgreSQL."
- "The request **was rejected** by the server."
- "Tests **should be written** before implementation."

### Gerunds vs. Infinitives
- **Gerund (-ing) after**: enjoy, avoid, consider, finish, suggest, keep, mind
  - "I suggest **refactoring** this module."
- **Infinitive (to) after**: want, need, decide, plan, expect, hope, agree
  - "I need **to fix** this bug."
- **Both (different meaning)**: stop, remember, try, forget
  - "I stopped **debugging**" (quit doing it) vs. "I stopped **to debug**" (paused to do it)

### Reported Speech
- Direct: He said, "I will fix it."
- Reported: He said he **would** fix it. (tense shift back one step)
